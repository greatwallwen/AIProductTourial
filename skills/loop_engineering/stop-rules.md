# 停机规则（写进项目根约定，所有 Agent 可见）

## 停止条件
1. ALL GREEN：所有检查通过，附每项通过证明。
2. 轮次用尽：达 5 轮上限，报告仍失败项、每轮尝试、为何没成。
3. 同一失败连续两轮：builder 在猜不在修，停止升级。
4. 回归：修复导致之前通过的检查失败，停止并说明。
5. 无实质进展：连续 2 轮失败项未减少，可能任务过大需拆分。
6. 疑似超出边界：失败涉及无法访问的外部依赖/环境，停止报告阻塞点。
7. 预算触顶：时间、token、费用或工具调用任一达到任务简报中的上限，暂停并请求批准。
8. 人工关口：涉及发布、外部写入、权限扩大或高影响决定时，序列化状态并等待明确批准。

## 红线
- 永不在没有 checker 输出时报告成功；永不弱化/删除/跳过检查换 ALL GREEN；永不修改 checker 的只读工具白名单。

## 升级协议
停止升级时必须携带：run id、当前轮次(Cycle N/5)、仍失败项与失败指纹、每项已尝试的修复、证据引用、已消耗预算、你的判断（为何继续循环不会解决）、建议由谁拍板。
