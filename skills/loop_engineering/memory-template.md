# 工作记忆区 · 通用模板（工具无关）

只存"未来仍有复用价值"的信息，不存完整聊天原文，不存凭证/令牌/身份等敏感数据。三层：
- `START-HERE`：入口说明（如何用、先读什么、什么不能写：只写长期有效、简短具体可人工核对、禁敏感数据）。
- `NOW`：当前工作台（进行中事项、卡点、下一步动作）。
- `DECISIONS`：重要决定，每条含 决定/原因/影响/**重新检查的条件**。
- 附：`contexts/`（事项稳定背景）、`playbooks/`（反复用的流程）、`preferences/`（长期协作偏好）、`inbox/`（待归类）。

## 就地说明（项目级，回答"这件事怎么做"）
每个项目根放一份项目说明：目标（一句话交付什么）、已确认边界（可改/不可改/须先确认）、验收标准（检查什么/输出什么/哪些文件别动）、记忆同步（收工把新决策/未解问题/可复用流程同步到工作记忆区）。

## 收工动作（最关键）
每次完成重要任务：提炼长期有效信息 → 写入决策记录（补原因与影响）→ 更新当前事项（只留未解与下一步）→ 可复用方法进流程区 → 不确定的进待整理区、不写成既定事实 → 不存聊天原文与敏感信息 → 说明改了哪些文件。
