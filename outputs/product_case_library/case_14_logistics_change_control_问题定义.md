# 需求变更影响评估（实操 14·问题定义）

> 数据来源：`dataset/product_cases/aicourse_logistics_delivery.csv`（700 行，异常 541）。字段与指标均回到该数据。演示原理 3.4、4.1，采用设计 amber-funnel。

## 产品问题

物流调度 PM 面对 700 张运单，要判断哪些线路超时、哪些城市该扩仓派单。异常类型（超时/破损/地址异常）散在运单里，扩城决策却要综合超时率、城市单量、实际时效，不能靠感觉。

## 岗位与业务对象

- 岗位：物流产品经理
- 业务对象：物流异常派单扩城
- 行业：物流配送

## 指标链（取自真实数据）

- 运单数：700
- 异常率：77.3%
- 扩城率：23.6%
- 城市数：8
- 平均实际时效(h)：62.96

## 异常状态与责任

- [改派] 上海 / — / 改派 → 责任 客服
- [缺车] 杭州 / — / 缺车 → 责任 网点
- [破损] 广州 / — / 破损 → 责任 干线
- [破损] 沈阳 / — / 破损 → 责任 客服
- [改派] 北京 / — / 改派 → 责任 干线
- [改派] 沈阳 / — / 改派 → 责任 网点

## 决策动作

判断漏斗断点、回款风险和区域销售动作优先级。

## 风险边界

不得在无回滚条件下扩大范围

## 使用 Skill

change-impact、release-risk、rollback-plan
