# 漏斗与预算建议（实操 31·问题定义）

> 数据来源：`dataset/reference_data_analysis/18-ad_performance.csv`（30 行，异常 10）。字段与指标均回到该数据。演示原理 3.2、4.3，采用设计 amber-funnel。

## 产品问题

投放 PM 每周复盘各渠道漏斗：曝光→点击→转化→成本。6 个渠道里社交曝光最大但 CVR 未必高，信息流 CPA 可能更优。预算要按真实转化效率重新分配，而不是按曝光量。

## 岗位与业务对象

- 岗位：投放产品经理
- 业务对象：广告投放转化复盘
- 行业：广告投放

## 指标链（取自真实数据）

- 渠道数：6
- 总曝光：4383576
- 总点击：177419
- 总转化：12870
- 平均CPA：46.13

## 异常状态与责任

- [转化率下滑] 品牌曝光 / 信息流A / 42999 → 责任 
- [CPA过高] 夏促 / 信息流A / 289667 → 责任 
- [创意点击低] 新品 / 信息流A / 110868 → 责任 
- [落地页转化低] 回流 / 信息流A / 114574 → 责任 
- [转化率下滑] 会员日 / 信息流A / 275916 → 责任 
- [CPA过高] 品牌曝光 / 信息流B / 254358 → 责任 

## 决策动作

定位各渠道漏斗断点（曝光/点击/转化），把预算从量大挪到效率高的渠道。

## 风险边界

不得只按点击率做预算决策
