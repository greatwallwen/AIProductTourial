# 案例 01 数据集设计：电商早会异常订单台

**文件**：`dataset/order_data.csv`（**真实基座 UCI Online Retail II（CC BY 4.0）** + 本地化改写实体标签 + 标注合成叠加；构建期零联网、确定性、可复现）
**真集来源**：UCI Online Retail II（一家英国在线零售商 2009–2010 年真实订单，CC BY 4.0）
**Schema**：电商订单（订单号/日期/SKU/品类/区域/数量/单价/金额/毛利率/库存天数/异常原因/责任人）

## 字段性质（真实 / 本地化改写 / 教学合成 —— 三类显式区分）
| 列 | 性质 |
|---|---|
| 金额 / 数量 / 单价 / 日期 / SKU | 真实（UCI 原始数值） |
| 品类 | 本地化改写（英文类目 → 中国电商类目，映射改写、非原始事实） |
| 区域 | 本地化改写（UCI 国家 → 中国省份，确定性轮转分配、非原始地理） |
| 毛利率 / 库存天数 / 责任人 / 处理时限 | 教学合成叠加（源数据无成本，页面与 MANIFEST 已标注） |
| 异常原因（退货等） | 真实信号，教学过采样约 ×5（真实约 2%，见 MANIFEST） |

## 设计的数据故事
4500 单真实英国电商订单，按品类/区域聚合看收入结构，把真实退货挑出来当场早会派单。**注意两处 base-rate 陷阱**：① 英国营收最高只是**样本主场体量**（数据本就来自英国零售商），不是增长/经营强度信号——与案例02「放款率被规模混淆」同型；② 异常率是 ×5 教学过采样的结果，别当真实经营水平。

## 说明（红线）
真实 / 本地化改写 / 教学合成三类**显式区分**（见 `dataset/MANIFEST.md` 与 metricSpec）：真实数值不动、改写实体标注、合成列绝不当真实经营结论——**也绝不把真实基座说成「确定性合成」**（这是反方向的红线）。
