# 案例 02 数据集设计：航空会员价值运营（RFM）

**文件**：`dataset/reference_data_analysis/2-air_data.csv`（本地、确定性合成、可复现）
**场景**：航空公司会员运营 PM 要基于 RFM（最近一次消费 Recency / 消费频次 Frequency / 消费金额 Monetary）做会员分层与价值运营。

## 字段义
| 列 | 义 | 角色 |
|---|---|---|
| 会员号 | 唯一会员 ID | 主键 |
| 卡等级 | 普卡/银卡/金卡/白金 | 维度 |
| 最近乘机天数 | 距今多少天未乘机 | **R**（越大越流失） |
| 年飞行次数 | 一年乘机次数 | **F**（越大越忠诚） |
| 年消费 | 年度消费金额（元） | **M**（越大越高价值） |
| 分层 | RFM 综合分层（重要价值/重要保持/重要发展/一般/流失预警） | 结论 |
| 里程余额 | 累计里程 | 辅助 |

## 设计的数据故事（埋入的真实结构）
1. **分层分布**：约 8% 重要价值（高 F 高 M 低 R）、15% 重要保持、20% 重要发展、其余一般/潜在流失——符合会员金字塔。
2. **高价值流失预警（核心异常）**：一批 **M 高（年消费前 40%）但 R 大（久未乘机后 40%）** 的会员——过去高价值、正在流失，是运营最该干预的人群。demo 要把这批人单独拎出来。
3. **R×F 负相关趋势**：越久未乘机（R 大）的会员，年飞行次数（F）越低——用散点可见。

## 为什么这样设计
RFM 是会员运营最经典的方法论；数据必须能让 demo 真算出 R/F/M 分位、真分层、真识别「高价值流失」。所有 KPI/分层/预警均从上述真实列真算（见 `code/server/services/cases.ts` 的 `rfm()`），不是模板占位。
