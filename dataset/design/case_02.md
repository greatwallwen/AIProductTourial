# 案例 02 数据集设计：大陆 P2P 信贷·信用画像分层

**文件**：`dataset/reference_data_analysis/p2p_credit.csv`（由 `dataset/real/renrendai_p2p.csv` 归一化派生，构建期零联网、确定性、可复现）
**真集来源**：人人贷 P2P 借贷记录（Harvard Dataverse `doi:10.7910/DVN/C4RUDY`，**CC0-1.0**，**中国大陆**）
**场景**：P2P 信贷风控 PM 要按借款人信用画像分层，做差异化放款与人工复核。**标的 `放款成功` = 是否融到款，非违约**——不可据此推断还款能力。

## 字段义
| 列 | 义 | 角色 | 性质 |
|---|---|---|---|
| 借款号 | 匿名借款 ID | 主键 | 真实（匿名） |
| 放款成功 | 是否融到款（1/0） | **标的**（≠违约） | 真实 |
| 借款金额 | 由 `Amount(log)` 还原 ¥ | 维度 | 真实（log 还原） |
| 学历等级 | 0–4 序数（**数据字典未给学历名，不臆造**） | 维度 | 真实 |
| 历史成功次数 | 历史成功借款次数 | 信用信号 | 真实 |
| 授信额度 | 由 `credit limit(log)` 还原 ¥ | 信用信号 | 真实（log 还原） |
| 有征信报告 | 有/无 | 信用信号 | 真实 |
| 描述字数 / 有效字符率 / 复杂词比例 / 主题1-4 | 借款描述文本特征 | 文案信号 | 真实 |
| 信用画像 | 优质/成长/待观察/薄档 | 分层结论 | **规则派生·非事实标签** |
| 风险信号 | 高风险薄档/需人工复核/空 | 异常队列 | **规则派生** |

## 真实结构与派生规则
1. **放款成功率 ≈ 20.6%**（真实基率）；有征信/无征信、不同信用画像层之间放款率差异显著。
2. **信用画像分层（规则派生）**：`score =（有征信?2:0）+ min(历史成功次数,2) +（授信额度≥中位?1:0）`，切 `优质(≥4)/成长(=3)/待观察(=2)/薄档(<2)`——**规则派生、非平台事实标签，已在 MANIFEST 与正文标注**。
3. **风险队列**：薄档 + 待观察（`风险信号` 非空）→ 高影响金融走人工复核，不自动放款。
4. **反直觉真实信号（混淆教材）**：放款成功率沿信用画像层是**反**的——薄档 52% > 优质 6%，无征信(32%)>有征信(16%)，额度≥5万仅 1.5% 融到款。真正驱动放款的是**借款规模**（小额易融），信用信号被规模压过。短文案放款率更高亦同理（短文案≈小额）。**放款成功≠信用好≠还得起**，相关≠因果。

## 为什么这样设计
干净可 vendored 的**大陆**信贷真集里**没有违约标的**（魔镜杯/拍拍贷/天池违约集均需账号且禁再分发），唯一 CC0 直连的人人贷标的是「放款成功」。故本案诚实地做**放款转化 + 信用画像分层**，而非违约预测——所有 KPI/分层/风险队列均从真实列真算或规则派生（见 `code/server/services/cases.ts` 的 `creditSegment()`），派生项全部标注。`periods` 列为归一化异常值，弃用以免误导。
