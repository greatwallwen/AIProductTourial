## 结课 Capstone：一条跨案例的端到端任务

把「底座 → 检索 → 分层 → 经营方案」串成一次完整的产品闭环——这正是把 L0–L4 能力地图走一遍：

> **任务**：为一家零售企业做一份「会员经营改进方案」。
> 1. **建库（案例 45）**：把经营数据落成关系库，能按区域/品类秒查（真实 SQL + 索引）。
> 2. **检索（案例 44）**：用 RAG 从产品知识库里，只召回与「会员流失」最相关的几段方法，喂给分析。
> 3. **分层（案例 30）**：用 RFM 把会员真分层，揪出「过去高价值、近期久未乘机」的高价值流失群。
> 4. **成方案（案例 41）**：把结论写成「现状 → 问题 → 动作 → 责任人 → 验收标准」的可交付方案，每条都能被真实数据验证。
>
> **验收**：方案里每个数字都能溯源到真实数据；每个动作都有责任人和可执行的验收条件；凡是教学合成的列（如毛利率），都明确标注、不当真实结论。这就是本书反复强调的「诚信」与「可验收」。

## 跨角色收口：同一个操作模型的三张脸（案例 48 / 49 / 50 / 41）

上面那份零售方案是「产品镜头」的闭环。但同一套操作模型，换个角色看是另一张脸——本书最后三个 dogfood 案例，就是让你在**同一份真实数据（本书自己的仓库）**上，把三个镜头一次看全：

> **任务**：给「本书这个项目」自己做一次交付评审，三镜头各出一张表。
> 1. **研发镜头（案例 48 · CI 失败分诊）**：CI 一红，是哪个接口契约、哪类问题、谁来修？——把「红了」变成结构化的分诊队列（真读本仓库的测试断言与接口契约）。
> 2. **产品镜头（案例 49 · RAG 评测台）**：这套问答「答得准不准」？——用标注评测集算覆盖命中率，把没答好的问题挑出来补语料（真读语料 × 评测集）。
> 3. **项目镜头（案例 50 · 交付门禁看板）**：到底「能不能发布」？——把每一项检查、每一条守卫列成可自动核对的门禁，红一项就发不了（真读本书的 verify 门禁）。
>
> **看穿它**：三张表其实是**同一块门禁的三张脸**——研发看代码断言、产品看回答质量、项目看发布关卡，都是把「对不对 / 好不好 / 能不能发」变成一组可自动、可复现的断言。这就是「一个操作模型，三个角色镜头」最实的证据；而案例 41 把三者收成一份能交付的经营方案。

## 用方法论建一个系统（不是几个 prompt）

学完架构章（§3）你就明白：一个中大型信息系统，不是几个 prompt 堆出来的。想真上手，挑一个你要做的系统，照 **SDD 八步**走一遍——宪法(`rules/`)→规格→澄清→架构(C4/DDD/ADR)→任务分解→逐任务 Loop→门禁三绿→演进。本书旗舰**案例 51「SDD 系统建造走查」**就是把「本平台自己」这么建出来的活样例，每一步都对得上仓库里一份真实工件；而**零售数字化中台**（§3.7）演示了一个大系统怎么拆成许多案例子系统。记住那句话：**方法论（分步）+ 门禁（三绿），两条腿才走得动一个大系统。**

## 结业与下一步

- **🎯 结业硬任务 · 带上你自己的项目走一遍**：别停在读完。挑一个你手头**真实**的系统或需求（例如「给我们产品的 AI 问答做一套 evals 并上线」），交出四样东西——① 一句话问题定义 + 一个可量化的北极星指标（§1 / §5 镜头）；② 一份最小规格 `spec.md`（§3 SDD 八步的前两步）；③ 一个能验收的「传感器」，哪怕只是一组 10 条测试题的 evals（§2「先建 Inspector 再建 Loop」）；④ 一条 L0→L3 上线分级 + 门禁（§6）。把这四样做出来，你就把本书从「知道」推到了「做到」。这也正是竞品 evals 课「用你自己的生产数据做作业」的精神。
- **深挖工程规范**：`rules/`（对齐 Google Style、SWE at Google、OWASP、12-Factor，出处见 `rules/references.md`）。
- **复用 Loop 实操**：`skills/loop_engineering/`（builder/checker/orchestrator/stop-rules/memory，工具无关，可套任意 Agent）。
- **记住一句话**：AI 时代产品经理的护城河，不是会不会写 prompt，而是懂不懂把「判断」沉淀成可复用、可验收的工程标准——并且，**永远诚实地区分真实与合成**。

> **一个学习提醒（间隔练习）**：合上书，别急着开新章。三天后回来，凭记忆默写这三样：三层 Loop 各在什么尺度、L0→L3 上线分级、休谟的归纳问题为什么逼你「验证而非证明」。想不起来的地方，就是你真正要回看的地方——这个「隔几天再取一次」的动作（间隔 + 检索），比再读一遍有效得多（回到导读「怎么真学会」）。
