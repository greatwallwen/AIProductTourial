## 结课 Capstone：一条跨案例的端到端任务

把「底座 → 检索 → 分层 → 经营方案」串成一次完整的产品闭环：

> **任务**：为一家零售企业做一份「会员经营改进方案」。
> 1. **建库（案例 05）**：把经营数据落成关系库，能按区域/品类秒查（真实 SQL + 索引）。
> 2. **检索（案例 04）**：用 RAG 从产品知识库里，只召回与「会员流失」最相关的几段方法，喂给分析。
> 3. **分层（案例 03）**：用真实 RFM（UCI 客户级真算）把会员分层，揪出「消费高（M 大）、近期久未复购（R 大）」的高价值流失群。
> 4. **成方案（案例 03）**：把结论写成「现状 → 问题 → 动作 → 责任人 → 验收标准」的可交付方案，每条都能被真实数据验证。
>
> **验收**：方案里每个数字都能溯源到真实数据；每个动作都有责任人和可执行的验收条件；凡是教学合成的列（如毛利率），都明确标注、不当真实结论。这就是本书反复强调的「诚信」与「可验收」。

**验收别只靠自觉——接上确定性传感器。** 把上面这份方案写成 markdown 后，先让机器替你过一遍确定性下限，再说「做完了」。本书的读者自测器 `check_my_work.mjs` 正是为这一步造的（§5、§8 用过它守案例），结课这里接上：把方案存成文件，跑

```bash
node code/tools/check_my_work.mjs 02 我的方案.md
```

它逐项核对你有没有把该案的字段、指标链、异常状态、所用 Skill、决策动作都落到位（红项会直接告诉你回哪一章补）。拿本书自带的范例交付物 `outputs/product_case_library/case_02_p2p_credit_方案验收.md` 当「我的方案」试跑一次，六项全绿：

```
自测 · 案例 02 p2p_credit · outputs/product_case_library/case_02_p2p_credit_方案验收.md

  ✔ 字段
  ✔ 指标链
  ✔ 异常状态
  ✔ Skill
  ✔ 决策动作
  ✔ 风险边界

✅ 全过：你的方案覆盖了本案的可核对要素（内容好坏仍需人判断——这只是确定性下限）。
```

记住最后那句括号里的话：**全绿只是「可核对要素齐了」的下限，内容好不好仍要人判断**——这正是全书那条「工具可以自动，判断不能外包」的红线，在结课这一刻的样子。

## 用方法论建一个系统（不是几个 prompt）

学完架构章（§3）你就明白：一个中大型信息系统，不是几个 prompt 堆出来的。想真上手，挑一个你要做的系统，照 **SDD 八步**走一遍——宪法(`rules/`)→规格→澄清→架构(C4/DDD/ADR)→任务分解→逐任务 Loop→门禁三绿→演进。本书旗舰**案例 08「SDD 系统建造走查」**就是把「本平台自己」这么建出来的活样例，每一步都对得上仓库里一份真实工件；而**零售数字化中台**（§3.7）演示了一个大系统怎么拆成许多案例子系统。记住那句话：**方法论（分步）+ 门禁（三绿），两条腿才走得动一个大系统。**

## 结业与下一步

- **🎯 结业硬任务 · 带上你自己的项目走一遍**：别停在读完。挑一个你手头**真实**的系统或需求（例如「给我们产品的 AI 问答做一套 evals 并上线」），交出五样东西——① 一句话问题定义 + 一个可量化的北极星指标（§1；写不清就用 §10.5 的 `requirement-grill` 让 Agent 反向面试你一轮）；② 一份最小规格 `spec.md`（§3 SDD 八步的前两步）；③ 一个能验收的「传感器」，哪怕只是一组 10 条测试题的 evals（§2「先建 Inspector 再建 Loop」）；④ 一条 L0→L3 上线分级 + 门禁（§5）；⑤ 把方案跑一遍 `node code/tools/check_my_work.mjs <最接近的案例号> 你的方案.md`，确认字段/指标链/Skill/决策动作等确定性要素不缺——过了下限，再请人对内容拍板。把这五样做出来，你就把本书从「知道」推到了「做到」。这也正是竞品 evals 课「用你自己的生产数据做作业」的精神。
- **深挖工程规范**：`rules/`（对齐 Google Style、SWE at Google、OWASP、12-Factor，出处见 `rules/references.md`）。
- **复用 Loop 实操**：`skills/loop_engineering/`（builder/checker/orchestrator/stop-rules/memory，工具无关，可套任意 Agent）。
- **记住一句话**：AI 时代实践者的护城河，不是会不会写 prompt，而是懂不懂把「判断」沉淀成可复用、可验收的工程标准——并且，**永远诚实地区分真实与合成**。

> **一个学习提醒（间隔练习）**：合上书，别急着开新章。三天后回来，凭记忆默写这三样：三层 Loop 各在什么尺度、L0→L3 上线分级、休谟的归纳问题为什么逼你「验证而非证明」。想不起来的地方，就是你真正要回看的地方——这个「隔几天再取一次」的动作（间隔 + 检索），比再读一遍有效得多（回到导读「怎么真学会」）。
